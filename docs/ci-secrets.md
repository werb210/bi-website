# CI secrets and variables

The Cleanup SWA Environments workflow needs Azure credentials to delete
preview environments when a PR closes. It runs gracefully (warning,
not failure) if these are absent.

## Option 1 — AZURE_CREDENTIALS (simplest)

Repo Settings → Secrets and variables → Actions → New repository secret

  Name: AZURE_CREDENTIALS
  Value (single JSON line):
    {"clientId":"…","clientSecret":"…","subscriptionId":"…","tenantId":"…"}

Generated by `az ad sp create-for-rbac --name bi-website-ci \
  --role contributor --scopes /subscriptions/<id>/resourceGroups/<rg> \
  --sdk-auth`.

## Option 2 — OIDC federated credentials (recommended)

Repo Settings → Secrets and variables → Actions

  AZURE_CLIENT_ID         (secret)
  AZURE_TENANT_ID         (secret)
  AZURE_SUBSCRIPTION_ID   (secret)

Plus configure the federated credential on the Azure AD app to trust
the GitHub repo + branch. See https://github.com/Azure/login#readme.

## Required repository variables (both modes)

Settings → Secrets and variables → Actions → Variables tab

  AZURE_SWA_NAME          e.g. boreal-bi-website
  AZURE_RESOURCE_GROUP    e.g. boreal-bi-rg

If either is missing the cleanup step warns and exits cleanly.
